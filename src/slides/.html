How various formats can deal with LaTeX math

HPL

Jan 26, 2013

This document is translated to the format pandoc.

Test 1: Inline math. A sentence contains the equation \(u(t)=e^{-at}\).

Test 2: A single equation without label. Here it is

\[ u(t)=e^{-at} \]

Test 3: A single equation with label. Here it is as a one-line latex code,

!bt
\begin{equation} u(t)=e^{-at} \label{eq1}\end{equation}
!et

looking like

\[ \begin{equation} u(t)=e^{-at} \label{eq1}\end{equation} \] and as a three-line latex code:

!bt
\begin{equation}
u(t)=e^{-at} \label{eq1}
\end{equation}
!et

looking like

\[ \begin{equation} u(t)=e^{-at} \label{eq1} \end{equation} \] This equation has label Equation (eq1).

Test 4: Multiple, aligned equations without label. Only the align environment is supported by other formats than LaTeX for typesetting multiple, aligned equations. The code reads

!bt
\begin{align*}
u(t)&=e^{-at}\\ 
v(t) &= \frac{du}{dt}
\end{align*}
!et

and results in

\[ \begin{align*} u(t)&=e^{-at}\\ v(t) &= \frac{du}{dt} \end{align*} \]

Test 5: Multiple, aligned equations with label. We use align with labels:

!bt
\begin{align}
u(t)&=e^{-at}
\label{eq2}\\ 
v(t) &= \frac{du}{dt}
\label{eq3}
\end{align}
!et

and results in

\[ \begin{align} u(t)&=e^{-at} \label{eq2}\\ v(t) &= \frac{du}{dt} \label{eq3} \end{align} \] We can refer to the last equations as the system Equations (eq2)-(eq3).
