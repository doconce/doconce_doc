 4  What Is Doconce? 
 5  ================ 
 6   
 7   
 8  Doconce is two things: 
 9   
10   1. Doconce is a very simple and minimally tagged markup language that 
11      looks like ordinary ASCII text (much like what you would use in an 
12      email), but the text can be transformed to numerous other formats, 
13      including HTML, Wiki, LaTeX, PDF, reStructuredText (reST), Sphinx, 
14      Epytext, and also plain text (where non-obvious formatting/tags are 
15      removed for clear reading in, e.g., emails). From reStructuredText 
16      you can go to XML, HTML, LaTeX, PDF, OpenOffice, and from the 
17      latter to RTF and MS Word. 
18      (An experimental translator to Pandoc is under development, and from 
19      Pandoc one can generate Markdown, reST, LaTeX, HTML, PDF, DocBook XML, 
20      OpenOffice, GNU Texinfo, MediaWiki, RTF, Groff, and other formats.) 
21   2. Doconce is a working strategy for never duplicating information. 
22      Text is written in a single place and then transformed to 
23      a number of different destinations of diverse type (software 
24      source code, manuals, tutorials, books, wikis, memos, emails, etc.). 
25      The Doconce markup language support this working strategy. 
26      The slogan is: "Write once, include anywhere". 
27   
28  What Does Doconce Look Like? 
29  ============================ 
30   
31  Doconce text looks like ordinary text, but there are some almost invisible 
32  text constructions that allow you to control the formating. For example, 
33   
34    - bullet lists arise from lines starting with an asterisk, 
35    - I{emphasized words} are surrounded by asterisks, 
36    - B{words in boldface} are surrounded by underscores, 
37    - words from computer code are enclosed in back quotes and 
38      then typeset verbatim (monospace font), 
39    - section headings are recognied by equality (C{=}) signs before 
40      and after the text, and the number of C{=} signs indicates the 
41      level of the section (7 for main section, 5 for subsection, 
42      3 for subsubsection), 
43    - paragraph headings are recognized by a double underscore 
44      before and after the heading, 
45    - blocks of computer code can easily be included by placing 
46      C{!bc} (begin code) and C{!ec} (end code) commands at separate lines 
47      before and after the code block, 
48    - blocks of computer code can also be imported from source files, 
49    - blocks of LaTeX mathematics can easily be included by placing 
50      C{!bt} (begin TeX) and C{!et} (end TeX) commands at separate lines 
51      before and after the math block, 
52    - there is support for both LaTeX and text-like inline mathematics, 
53    - tables, figures, movies with captions, URLs with links, index list, 
54      labels and references are supported, 
55    - comments can be inserted throughout the text (C{#} at the beginning 
56      of a line), 
57    - with a simple preprocessor, Preprocess or Mako, one can include 
58      other documents (files) and large portions of text can be defined 
59      in or out of the text, 
60    - with the Mako preprocessor one can even embed Python 
61      code and use this to steer generation of Doconce text. 
62   
63  Documentation of Doconce is found in 
64   
65    - The tutorial in C{doc/tutorial/tutorial.do.txt} (file paths are here given 
66      relative to the root of the Doconce source code). 
67    - The more comprehensive documentation in C{doc/manual/manual.do.txt}. 
68    - There is web access to the U{tutorial<http://doconce.googlecode.com/hg/doc/demos/tutorial/index.html>} and the U{manual<http://doconce.googlecode.com/hg/doc/demos/manual/index.html>} 
69   
70  Both directories contain a C{make.sh} file for creating various formats.
